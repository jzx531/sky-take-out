# “苍穹外卖” Java企业级项目实战

## 📅 第一阶段：项目启动与基础数据维护（第1-4天）

这一阶段主要目标是跑通项目环境，并完成后台管理系统的基础CRUD（增删改查）功能。

### Day 1：项目概述与环境搭建
内容：了解软件开发生命周期、项目需求分析、技术选型（Spring Boot, MyBatis, MySQL, Redis等）。
任务：搭建开发环境（JDK, Maven, Git, IDEA），导入前后端代码骨架，配置数据库，实现后端与数据库的连接，完成第一个接口测试（如Swagger集成）。
重点：理解分层架构（Controller, Service, Mapper），熟悉Git协作流程。

### Day 2：员工管理与分类管理
内容：实现后台管理系统的核心基础模块。
任务：
员工管理：登录认证（JWT令牌）、新增员工、分页查询、启用/禁用账号、编辑员工信息。
分类管理：菜品分类和套餐分类的新增、修改、删除、分页查询及根据类型查询。
重点：拦截器/过滤器实现登录检查、ThreadLocal存储用户ID、全局异常处理。

### Day 3：菜品管理
内容：处理复杂的菜品数据逻辑。
任务：新增菜品（涉及文件上传阿里云OSS、多表关联插入）、菜品分页查询、修改菜品（回显数据、更新口味）、删除菜品（逻辑判断是否被套餐关联）。
重点：文件上传、多表操作事务控制、DTO与VO的设计。

### Day 4：套餐管理（项目实战）
内容：综合应用前几天的知识，完成套餐模块。
任务：新增套餐（关联菜品）、套餐分页查询、修改套餐、启售/停售套餐、删除套餐。
重点：强化多表关联操作能力，理解套餐与菜品的关系（一对多）。

## 📅 第二阶段：C端点餐业务核心开发（第5-10天）

这一阶段是项目的核心，模拟用户微信小程序端的完整点餐流程，涉及高并发、缓存、支付等高级技术。

### Day 5：店铺营业状态与微信登录
内容：控制店铺状态，实现用户身份认证。
任务：
店铺状态：管理员设置营业/打烊状态，用户端根据状态限制下单。
微信登录：对接微信接口，获取OpenID，实现自动登录/注册，生成JWT令牌。
重点：微信API调用、HTTP客户端工具使用、自定义登录逻辑。

### Day 6：商品浏览与缓存优化
内容：用户端首页展示，引入Redis提升性能。
任务：展示分类列表、展示菜品/套餐列表。
重点：Redis缓存应用（缓存菜品数据），解决频繁查库问题，缓存更新策略（修改菜品时同步删除缓存）。

### Day 7：购物车功能
内容：实现用户的临时数据存储。
任务：添加购物车、查看购物车、清空购物车、减少数量。
重点：利用Redis Hash结构存储购物车数据，区分不同用户的购物车，处理菜品估清逻辑。

### Day 8：用户下单与支付
内容：核心交易链路。
任务：
下单：确认订单信息（地址、金额计算）、扣减库存、保存订单数据。
支付：对接微信支付沙箱环境，生成支付二维码/参数，处理支付回调。
重点：订单状态机设计、分布式ID生成（雪花算法）、事务控制、支付安全校验。

### Day 9：历史订单与订单管理
内容：订单的生命周期管理。
任务：
用户端：查看历史订单、订单详情、再来一单、取消订单。
管理端：订单搜索、各状态订单统计、接单/拒单、派送、完成订单。
重点：复杂SQL查询、订单状态流转控制。

### Day 10：定时任务与消息提醒
内容：自动化处理与实时交互。
任务：
定时任务：使用Spring Task处理“超时未支付订单”自动取消、“一直处于派送中”的订单自动完成。
消息提醒：实现来单语音提醒（WebSocket）、客户催单功能。
重点：Spring Task定时任务、WebSocket全双工通信、异步处理。

## 📅 第三阶段：数据统计与项目总结（第11-12天）

最后一阶段侧重于运营数据的可视化展示，提升项目的商业价值感。

### Day 11：数据统计（图形报表）
内容：为管理者提供数据看板。
任务：
工作台：展示今日数据（营业额、有效订单、完成率等）、订单管理概览。
图表数据：查询指定时间段内的营业额趋势图、用户总量趋势图、热销菜品排行。
重点：复杂聚合SQL查询、ECharts前端图表对接、日期范围处理。

### Day 12：数据导出与项目部署
内容：报表导出与最终交付。
任务：
Excel导出：使用Apache POI技术，将运营数据导出为Excel报表。
项目总结：代码重构、常见面试题梳理、项目部署（Nginx反向代理、Linux环境部署）。
重点：Apache POI操作Excel、Nginx配置、项目上线流程。

💡 学习建议
不要只抄代码：重点理解业务逻辑（如订单状态如何流转）和技术选型原因（为什么用Redis存购物车）。
注重调试：在开发支付和微信登录时，沙箱环境和真实接口容易出错，学会看日志和断点调试。
扩展思考：学完后可以尝试优化，例如引入RabbitMQ处理订单超时（替代定时任务轮询），或引入Elasticsearch做菜品搜索。
